# scope.md — xiaohongshu-trend-agent

## §1 IN（做什么）

| # | 功能 | 验收标准 |
|---|------|----------|
| 1 | 接收品类关键词 | 支持中文关键词输入，如"美妆"、"母婴"、"家居" |
| 2 | 检索公开热门笔记 | 调用数据工具返回 Top N 热门笔记元数据 |
| 3 | 分析爆文共性要素 | LLM 提取：标题模式、封面规律、内容结构、互动特征 |
| 4 | 输出结构化报告 | Markdown 格式，含分类维度 + 具体案例引用 |
| 5 | 对话式追问 | 用户可追问细化（如"封面用什么配色"） |

## §2 OUT（不做什么）

| 能力 | 为什么不做 |
|------|-----------|
| 自动发布笔记 | 超出分析范围，涉及账号安全 |
| 爬取私域数据 | 合规红线 |
| 竞品账号深度画像 | 涉及个人隐私 |
| 实时监控/定时任务 | MVP 阶段仅对话式 |
| 多平台分析（抖音/微博） | 聚焦小红书单平台 |

## §3 Feature Amendment 门槛

新增任何功能前必须满足：

1. 不违反 Compliance 红线（不爬私域、不存 ID、不造假）
2. 不引入非必要的 LLM 依赖
3. 不扩大数据获取边界（仅公开数据）
4. 个人博主用户有明确需求表达
