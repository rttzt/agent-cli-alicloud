# 澄清问题与答案

> Agent 想法：我想做一个能帮品牌方分析小红书爆文的 Agent

## 1. 终端用户

**问题**: 这个 Agent 的目标用户是谁？

**答案**: 品牌方运营人员和内容策划，需要快速了解小红书内容趋势

## 2. 触发方式

**问题**: 用户如何触发/启动这个 Agent？

**答案**: 通过输入品类关键词或竞品账号名称来触发分析

## 3. 数据来源

**问题**: Agent 需要什么数据输入？来源是什么？

**答案**: 小红书公开笔记数据，通过小红书搜索 API 或第三方爬虫获取（用户自行配置 API Key）

## 4. 输出形态

**问题**: Agent 的输出是什么形式？

**答案**: 结构化的爆文要素拆解报告（标题、封面、钩子、标签、CTA）+ 选题建议清单

## 5. 合规要求

**问题**: 有什么数据安全或合规方面的要求？

**答案**: 仅采集公开可见内容，不存储用户个人 ID，不进行任何形式的数据倒卖
